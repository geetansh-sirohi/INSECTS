# 🪲 INSECT — Interactive Insect Landing Page

A fully interactive 2-slide insect showcase built using **only HTML & CSS — no JavaScript.**
Created as part of **Sheryians Coding School, Cohort 3.0** assignment.

---

## 🚀 What I Built

The original assignment was a simple single-page static design.
I took it further — converted it into a **2-slide interactive UI** with smooth transitions, hover animations, responsive layouts, and different color themes per slide.

Everything you see — every interaction, every animation — runs purely on CSS.

---

## ✨ Features

- **2-slide interactive layout** with CSS-only slide switching (checkbox hack)
- **Second insect added** — Black Widow, making it feel like a real multi-product site
- **Hover zoom effects** on insect images
- **Smooth CSS transitions** throughout
- **Fully responsive** — Desktop, Tablet, and Mobile
- **Different color themes** for each slide
- **Custom fonts** via Google Fonts + Font Awesome icons

---

## ⚠️ Real Challenges I Faced

**1. No Flexbox or Grid**
These weren't taught yet. I built every layout using only positioning, margins, paddings, and display properties — which gets surprisingly complex on smaller screens.

**2. No JavaScript allowed**
Every interactive element — slide switching, hover states, animations — was built using `:hover`, CSS transitions, and the checkbox trick. It forced creative thinking.

**3. Slide 2 kept breaking on mobile**
Even though both slides had similar structures, Slide 2's classification section kept shrinking and margins behaved inconsistently. I ended up writing completely separate responsive classes for it and rebuilding the spacing rules from scratch.

**4. Testing responsiveness properly**
Chrome DevTools alone wasn't enough. I used a Mobile Simulator extension and tested manually across multiple screen sizes.

**5. Time pressure**
I'm a Class 10 student — balancing board prep, school, and three assignments simultaneously. This was built under a tight deadline.

## 📚 What I Learned

- How far you can push CSS without modern tools like Flexbox or Grid
- Why planning your layout structure *before* writing code actually matters
- Responsive design is hard — and that difficulty taught me more than easy projects would
- How reusable class structures make future pages much easier to build
- Most importantly: solving tough UI problems is genuinely enjoyable
